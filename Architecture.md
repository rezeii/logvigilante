The architecture implements a 'Strict Atomic Stream' pattern. 1. Validation: Pre-flight checks for directory write-bits and cross-device link potential. 2. Isolation: Temporary files are created with restricted (0600) permissions using mkstemp. 3. Streaming: Line-limited iteration prevents OOM, while a length-check prevents partial-match evasion. 4. Atomicity: A four-stage commit (Flush -> Fsync-File -> Copymode -> Replace -> Fsync-Dir) ensures zero-length files or ghost-metadata never persist after a crash. 5. Signal Safety: SIGINT/SIGTERM are trapped to set a volatile flag, ensuring the 'finally' blocks trigger clean filesystem removal of artifacts.
