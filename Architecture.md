1. Durability: Extends atomic writes by calling fsync on the parent directory. This prevents 'ghost' files where the file data is synced but the directory entry is lost on crash.
2. Security: Implements permission mirroring. The temporary file is created with 0600, but is updated via copymode to match the source file's metadata before the atomic swap.
3. Memory Safety: Replaced standard line iteration with a length-limited generator. This protects against 'Log Bombs' where a single line exceeds available RAM.
4. Signal Integrity: Added a global interrupt flag and handler to ensure the process cleans up temporary artifacts even during a Ctrl+C event.
