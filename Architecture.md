The codebase follows a 'Fail-Fast' philosophy. 1. **Initialization Layer**: Validates file existence, permissions, and compiles regex strings before any file handles are opened. 2. **Processing Layer**: Uses a context-manager pattern for safe resource handling. It employs a streaming buffer approach (`for line in fin`) to ensure O(1) space complexity relative to file size. 3. **Error Handling**: A hierarchy of custom exceptions (LogVigilanteError) separates user-input errors from system-level I/O failures, facilitating clearer logging and more predictable CLI behavior in automated pipelines.
