The architecture strictly adheres to the 'Strict Atomic Stream' pattern with enhanced safety layers: 1. Environmental Validation: Ensures target directories exist and checks for write-bits. 2. Secure Isolation: Employs mkstemp for 0600 file creation within the same filesystem to facilitate atomic renaming. 3. Memory-Bounded Streaming: Replaces standard line-iteration with `.readline(limit)` to prevent OOM before validation. 4. Four-Phase Commit: Data is flushed, then hardware-synced (fsync), followed by metadata preservation (copymode), and finally an atomic replacement. 5. Persistence Guarantee: Synchronizes the parent directory file descriptor to ensure the dentry update is durable across power cycles. 6. Signal-Resilient Cleanup: Uses volatile flags and state-aware 'finally' blocks to ensure zero orphaned temporary files on SIGINT/SIGTERM.
